# Frame A Discipline

- Watch token budget before work gets thin: use `ck:context-engineering` when a session is long, fragmented, or close to compaction.
- Cut clean phase boundaries: create a handoff or `watzup` report before switching contexts, not after details have already faded.
- Keep one task in one clean session: avoid mixing unrelated implementation, research, and review threads unless the plan explicitly couples them.
- Prefer committed source-of-truth artifacts over memory: plans, ADRs, learning records, and handoffs should point to real files and evidence.
- Review `ntq:handoff` vs `watzup` after roughly one week of use; keep the one that creates less repeat explanation.
